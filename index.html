A two-player game that never leaves the terminal. Names get typed in, moves get pressed in one keystroke, and every X, O, and win line gets its own ANSI color.

[ Play Now ] [ View Build Steps ]

Features

./tictactoe --features
  • ›Hot-seat two-player — both names are typed in at the start, then the board decides whose turn it is.
  • ›One keypress, no Enter — moves register the instant a key is hit, via termios on Linux/macOS and conio.h on Windows.
  • ›Full ANSI color board — Player X renders in red, Player O in blue, straight from the terminal's own color codes.
  • ›Every win line detected — rows, columns, and both diagonals are all checked after each move.
  • ›Winning line highlighted — the three winning squares redraw in green so the win is easy to spot.
  • ›Draw detection — nine moves with no winner prints a clean "Game Draw" screen.
  • ›Replay loop — win or draw, you're offered a rematch without restarting the program.

Build It Yourself

./build.sh
# compile
g++ -std=c++17 tictactoe.cpp -o tictactoe
 
# run
./tictactoe

On Windows the build pulls in <conio.h> for instant key reads; on Linux and macOS it falls back to a small termios-based getch(), so a single keypress moves the game forward the same way on every platform.
